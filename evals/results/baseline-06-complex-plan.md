Agor separates the UI, daemon, core library, and executor. A cross-cutting plan should first identify which component owns the behavior, then treat the others as callers or consumers. Shared types and invariants belong in core, durable service behavior belongs in the daemon, runtime-specific execution belongs in the executor, and presentation belongs in the UI. The implementation plan should change only the necessary owner and adapters. Validation should include focused deterministic tests plus an isolated live call through the real interface that crosses the changed boundary. Approval covers only the described local implementation and validation, not deployment or another external action.

The PR review experience should use one shared format for the initial review and its follow-up. Reviews should orient the author, identify what is good, and group actionable findings by severity. Each finding needs a descriptive title, verified location, reasoning, and a specific change. The follow-up should map addressed work back to the same severity groups and state validation and remaining issues. Organization profiles decide language and local conventions. Internal workflow details and local paths must stay private, and producing the draft does not authorize posting it.

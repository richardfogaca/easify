The archive-cascade fix needs evidence at several levels. A database assertion can show descendant rows changed, but it does not prove that authorization ran, that realtime events were emitted, or that a client observed the new state. The strongest regression first reproduces the original missing-descendant behavior on unchanged code and then passes after the fix. Service tests can prove authorization and event-hook behavior, while an isolated HTTP/WebSocket run is needed for live transport confidence. If that runtime cannot be exercised, the report should say transport evidence is degraded rather than infer it from typecheck or unit tests. All claims must cover the final checked state.
